# Contributing to AksaraLLM

First off, thank you for considering contributing to AksaraLLM! 🎉

It's people like you that make open source such a fantastic community to learn, inspire, and create. Whether you are a seasoned AI researcher, a beginner programmer, or a native speaker wanting to improve your language's representation, your contribution is highly valued.

This document serves as a set of guidelines for contributing to the AksaraLLM organization and its repositories.

## 🤝 Code of Conduct

By participating in this project, you are expected to uphold our [Code of Conduct](CODE_OF_CONDUCT.md). Please report any unacceptable behavior to our moderation team via Discord.

## 🚀 How Can I Contribute?

### 1. Reporting Bugs
Bugs are tracked as GitHub issues. When creating an issue, please use the provided `Bug Report` template. Explain the problem and include additional details to help maintainers reproduce the problem:
- **Use a clear and descriptive title** for the issue to identify the problem.
- **Describe the exact steps** which reproduce the problem.
- **Provide specific examples** to demonstrate the steps (e.g., provide code snippets or link to a gist).
- **Describe the behavior** you observed after following the steps and point out what exactly is the problem with that behavior.

### 2. Suggesting Enhancements
Enhancement suggestions are tracked as GitHub issues. When creating an issue, use the `Feature Request` template and provide the following:
- **Use a clear and descriptive title** for the issue.
- **Provide a step-by-step description** of the suggested enhancement.
- **Explain why this enhancement would be useful** to most AksaraLLM users/developers.

### 3. Curating Datasets
For language models, data is as important as code. You can contribute by:
- Translating or verifying existing datasets in our `dataset-pipeline` repository.
- Suggesting open-source public domain texts in your native language.
- Reviewing Pull Requests related to dataset curation for cultural context and linguistic accuracy.

### 4. Code Contributions (Pull Requests)
We love Pull Requests! If you want to dive into the codebase:
1. **Fork the repository** and clone your fork locally.
2. **Create a branch** for your native feature (`git checkout -b feature/amazing-feature`).
3. **Write tests** for your changes if applicable.
4. **Follow the code style** of the repository (we generally enforce PEP 8 for Python).
5. **Commit your changes** using descriptive commit messages.
6. **Push to the branch** (`git push origin feature/amazing-feature`).
7. **Open a Pull Request** using our PR template.

## ✔️ PR Review Process
- Ensure your code passes all continuous integration (CI) checks.
- A core maintainer will review your code. We may suggest some changes or improvements or alternative approaches.
- Once approved, your PR will be merged into the main branch.

Thank you for helping us democratize AI out of the language barriers! 🌍
