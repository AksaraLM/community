## 🏛️ Governance

AksaraLLM uses a **meritocratic governance model** inspired by the Apache Software Foundation. See [GOVERNANCE.md](GOVERNANCE.md) for details.

### Roles

| Role | Description | Access |
|------|-------------|--------|
| **Contributor** | Anyone who contributes | PR access |
| **Committer** | Regular, trusted contributor | Write access to repos |
| **Maintainer** | Leads a specific repo/area | Admin of specific repo |
| **Core Team** | Key decision makers | Org-wide admin |
| **Project Lead** | Overall direction | Full admin |

## 📝 RFC Process

For significant changes (architecture, data decisions, training methodology), we use an RFC (Request for Comments) process:

1. **Open an issue** using the RFC template
2. **Community discussion** for 1-2 weeks
3. **Core team vote** to accept/reject/modify
4. **Implementation** once accepted

See [rfcs/](rfcs/) for existing RFCs.
